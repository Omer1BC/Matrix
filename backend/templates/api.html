> Getting Started

Welcome to the Matrix API. This RESTful API allows you to integrate Matrix's technical interview preparation platform into your applications. All endpoints return JSON responses and use standard HTTP response codes.

Base URL

https://matrix-backend.up.railway.app

Response Format

All responses follow this standard structure:

{ "success": true, "data": { ... }, "message": "Operation successful", "timestamp": "2024-12-03T10:30:00Z" }

> Authentication

All API requests require authentication using an API key. Include your API key in the request header:

curl -H "Authorization: Bearer YOUR_API_KEY" \ -H "Content-Type: application/json" \ https://matrix-backend.up.railway.app/problems

You can generate an API key from your dashboard settings.

> Neo AI Agent

Interact with Neo, our AI assistant designed specifically for technical interview preparation.

POST /neo/chat

Send a message to Neo and receive intelligent assistance.

Request Body

{ "problem_id": "two-sum", "message": "I'm stuck on this problem. Can you give me a hint?", "context": { "current_code": "function twoSum(nums, target) { ... }", "user_approach": "I'm trying to use nested loops" } }
200 OK
{ "success": true, "data": { "neo_response": "Good start! Nested loops work, but consider: can you use a hash map to reduce time complexity?", "hint_level": 1, "follow_up_questions": [ "What is the time complexity of your current approach?", "What data structure allows O(1) lookups?" ] } }
POST /neo/analyze

Submit code for Neo to analyze and provide feedback on correctness, efficiency, and style.

GET /neo/hints/{problem_id}

Get progressive hints for a problem without spoiling the solution.

> Learning Modules

GET /learning/modules

Access structured learning paths for data structures and algorithms topics.

GET /learning/visualize/{algorithm}

Get visualization data for algorithm animations (quicksort, BFS, DFS, etc.).

> Rate Limits

API rate limits are enforced per API key:

Tier Requests/Hour Neo Interactions/Hour
Free 100 20
Pro 1,000 200
Enterprise Unlimited Unlimited

Rate limit information is included in response headers:

X-RateLimit-Limit: 1000 X-RateLimit-Remaining: 847 X-RateLimit-Reset: 1701619200

> Error Codes

Standard HTTP status codes are used:

Code Description
200 Success
400 Bad Request - Invalid parameters
401 Unauthorized - Invalid or missing API key
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error
